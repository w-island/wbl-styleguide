# wbL 스타일가이드 적용 방법

이 문서는 wbL 디자인 시스템을 **실제 작업물에 입히는 여러 방법**을 정리합니다.
**누구나(다른 팀·다른 PC 포함)** 따라 할 수 있게 설치·적용 절차를 담았습니다.
라이브 가이드: **https://w-island.github.io/wbl-styleguide/**
모든 토큰의 단일 원천은 Figma `wbL Primitives / Semantic` 입니다.

핵심 규칙 3가지(어떤 방법이든 공통):
1. **HEX 직접 쓰지 말고 토큰(CSS 변수)으로.**
2. **에디토리얼 미니멀** — 여백 넉넉히, 한 화면에 강조(CTA)는 하나.
3. **색은 시맨틱 토큰으로** → 다크 모드 자동 대응.

---

## 방법 1. Claude 스킬로 적용 (권장 · 누구나 설치 가능)

스킬 파일이 이 레포 안에 들어 있습니다 → **[`skill/wbl-design/`](skill/wbl-design/)**
(`SKILL.md` = 토큰·컴포넌트·카피 규칙, `tokens.css` = 드롭인 토큰)

### 1단계 — 설치 (각자 PC에서 1회)
Claude Code 사용자는 스킬 폴더를 자기 PC의 `~/.claude/skills/`(개인 스킬) 또는
프로젝트의 `.claude/skills/`(팀 공유) 아래에 복사하면 됩니다.

**A. git으로 (권장)**
```bash
git clone https://github.com/w-island/wbl-styleguide.git
# macOS/Linux
mkdir -p ~/.claude/skills && cp -r wbl-styleguide/skill/wbl-design ~/.claude/skills/
# Windows (PowerShell)
Copy-Item wbl-styleguide\skill\wbl-design "$HOME\.claude\skills\" -Recurse
```

**B. 파일만 내려받아 복사**
레포의 `skill/wbl-design/` 폴더(SKILL.md + tokens.css)를 통째로
`~/.claude/skills/wbl-design/` 에 넣습니다.

**C. 팀 레포에 상시 공유**
작업 레포의 `.claude/skills/wbl-design/`에 커밋해 두면, 그 레포를 여는 모든 팀원이 자동으로 사용합니다.

> 설치 후 Claude Code를 재시작(또는 새 세션)하면 `wbl-design` 스킬이 인식됩니다.

### 2단계 — 사용
Claude Code에서 아래처럼 말하면 스킬이 자동으로 걸립니다.
```
wbL 스타일로 상세페이지 히어로 섹션 만들어줘
이 랜딩 페이지 wbL 디자인 시스템으로 바꿔줘
wbL 톤으로 이벤트 배너 카드 3개 만들어줘
```
명시적으로 부를 수도 있습니다: `/wbl-design 상세페이지 히어로`
스킬을 쓰면 Claude가 토큰·컴포넌트 클래스·카피 규칙을 **알고서** 만들기 때문에, 매번 규칙을 설명할 필요가 없습니다.

### 스킬 갱신 (관리자)
가이드(`index.html`)의 토큰이 바뀌면 레포의 `skill/wbl-design/`(SKILL.md·tokens.css)를 갱신·커밋합니다.
사용자는 다시 내려받아 `~/.claude/skills/`에 덮어쓰면 됩니다.
(원천 우선순위: Figma → index.html → skill/)

## 방법 2. 프롬프트로 적용 (스킬 없이, ChatGPT/Gemini 등에서도)

스킬이 없는 환경(ChatGPT, Gemini, Figma AI 등)에서는 아래 **시스템 프롬프트**를 앞에 붙이세요.

### 붙여넣기용 시스템 프롬프트
```
너는 wbL(주얼리 브랜드) 디자인 시스템으로 UI를 만드는 디자이너다.
반드시 아래 규칙을 지켜라.

[무드] 에디토리얼 미니멀 · 세이지 그린/아이보리/차콜 · 여백 넉넉 · 절제된 럭셔리.

[색상 — HEX 직접 쓰지 말고 토큰으로]
- 브랜드 코어 green-400 #a5b398, 강조 버튼 green-700 #515e45
- 주 CTA는 차콜 #454342(흰 글씨), hover #2e2c2b
- 텍스트: 제목 #363636, 본문 #3e3948, 보조 #7c7c7c
- 배경: 기본 #ffffff, 아이보리 섹션 #e4e0d5, 보조면 #f7f7f7
- 라인/보더 #d7d7d7, 세일 #c0392b

[타이포] Pretendard. Display 48/700, H1 36/700, H2 28/600, H3 22/600,
Subtitle 18/500, Body 16/400, Caption 13/500, Price 18/700. 자간 제목 -2%.

[스페이싱/라운드] 4px 베이스(4·8·12·16·20·24·32·40·48·64).
radius sm8/md12/lg16/xl20/pill999. 그림자는 옅게만.

[아이콘] 얇은 라인 1.4~2px, 끝 둥글게, 차콜 라인 + 은은한 세이지 면.
별·마법·우주·의료차트·키치 금지.

[버튼 위계] 한 화면에 강조 CTA는 하나(차콜). 세이지=보조 강조,
아이보리/아웃라인=보조 행동.

[카피 보이스] "몇 초 만에 더하는 자신감(Confidence in Seconds)".
순간·즉각성 강조("몇 초","지금","바로"), 다른 루틴 존중하는 동행의 언어,
담백·단정. 금지: 효능 단정, 할인·최저가 언어, 과한 이모지/느낌표,
"~보다 낫다"식 비교.
```

그 뒤에 요청을 붙입니다:
```
위 규칙으로 [만들 대상]을 만들어줘. HTML+CSS로, 색은 CSS 변수로.
```

### 카피만 다듬을 때(짧은 프롬프트)
```
아래 문구를 wbL 보이스로 다듬어줘.
- 순간·즉각성 강조, 담백·단정, 동행의 언어
- 금지: 효능 단정 / 할인 언어 / 과한 이모지 / 비교
문구: "..."
```

## 방법 3. 코드에 직접 적용 (개발/퍼블리싱)

### 3-1. 드롭인 토큰 CSS
레포의 [`skill/wbl-design/tokens.css`](skill/wbl-design/tokens.css)(또는 `index.html`의 `:root` 블록)를 복사해
프로젝트 최상단에 넣고, 색은 전부 `var(--...)`로 씁니다.

```html
<link rel="stylesheet" href="tokens.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.min.css">
```
```css
.buy-btn{ background:var(--action-default); color:var(--action-on);
  border-radius:var(--r-sm); padding:13px 22px; }
```

### 3-2. 다크 모드
`<html data-theme="dark">`만 토글하면 시맨틱 토큰이 자동 반전됩니다.
(색을 HEX로 하드코딩하면 반전이 안 되니 반드시 토큰 사용)

### 3-3. 컴포넌트 재사용
`index.html`의 클래스를 그대로 씁니다: `.btn`(+`-cta/-brand/-secondary/-outline/-ghost`),
`.badge`, `.field/.input`, `.tabs/.tab`, `.chip`, `.alert`, `.card`, 아이콘(`.isf/.isd/.icf`).

## 방법 4. Cafe24 자사몰(wblbeauty.com)에 적용

Cafe24 SmartDesign 편집기는 붙여넣기 방식입니다.
- 공통 `<head>` 또는 상단 모듈에 `tokens.css`의 `:root` 블록 + Pretendard CDN을 넣습니다.
- 이후 상세/기획전 HTML에서 색·라운드·타이포를 토큰으로 씁니다.
- `{$...}` 토큰 포맷터와 CSS 변수는 충돌하지 않으니 그대로 사용 가능.

---

## 방법 5. Figma에서 적용

- Figma 라이브러리 `wbL Primitives / Semantic`이 **원천**입니다. 여기서 변수/스타일을 쓰세요.
- 코드↔Figma가 어긋나면 **Figma를 정답**으로 두고 `index.html`·스킬을 맞춥니다.

---

## 우선순위 · 동기화 규칙

```
Figma (원천)  →  index.html (라이브 가이드)  →  wbl-design 스킬 / tokens.css
```
- 새 값은 위에서 아래로 흐릅니다.
- 색을 바꿔야 하면 개별 요소가 아니라 **토큰 값**을 바꿉니다(화면 전체 일괄).
- 애매하면 라이브 가이드(https://w-island.github.io/wbl-styleguide/)를 눈으로 확인 — 스와치 클릭 시 HEX 복사됩니다.
